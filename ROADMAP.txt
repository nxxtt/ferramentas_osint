# MyTools Roadmap
1. Subfinder / passive DNS lookup
Atualmente subdomainenum é só brute-force DNS. Uma feature de passive subdomain enumeration usando fontes públicas (crt.sh, VirusTotal, SecurityTrails) encontraria muito mais subdomínios sem brute-force.

2. HTTP screenshot / visual diff
Tirar screenshot das páginas encontradas (via playwright headless) e comparar visualmente. Útil para identificar:
Portais de login ocultos
Diferenças entre ambientes (prod/staging)
Aplicações que mudam conteúdo dinamicamente

5. Screenshot + JS rendering para dirscanner
O dirscanner não renderiza JavaScript. SPAs retornam o mesmo shell para todos os paths (detectado por detect_spa_fallback). Usar Playwright para renderizar JS resolveria falsos negativos.

6. Subdomain takeover check
Verificar se subdomínios apontam para CNAMEs de serviços (GitHub Pages, Heroku, S3, etc.) que estão livres — vulnerabilidade comum e fácil de automatizar.

7. HTTP request smuggling básico
Testar CL.TE e TE.CL smuggling entre proxy e backend. Atualmente não há nada similar.

8. Credential leak em respostas
Varredura de padrões de credenciais em bodies: AWS keys, tokens JWT, private keys PEM, connection strings de banco. O attackaudit.py tem path probing mas não regex de credenciais.

9. Multi-wordlist support no subenum
Hoje suporta uma wordlist customizada. Suportar múltiplas wordlists e merge automático (内置 + custom) seria útil.

11. API server (REST)
Expor os módulos como API HTTP para integração com outros tools (Slack bots, dashboards, CI pipelines).

12. Geolocation de IPs
Enriquecer portscanner com geolocation dos IPs encontrados (usando db gratuita como MaxMind GeoLite2).

13. DNS over HTTPS (DoH) support
Alternativa ao DNS tradicional, útil em ambientes com DNS filtering.

14. WebSocket scanner
Testar endpoints WebSocket para vulnerabilities (CSWSH, cross-site WebSocket hijacking).
